SnapMD VirtualCare REST API

Introduction

It is our goal to keep improving our documentation and to keep it up-to-date. If you have any questions, please contact your SnapMD representative. If there are problems with the documentation, please report an issue on GitHub.

API Credentials

API credentials consist of a unique identifier and an SHA1 hash. The values must be passed in your requests via custom HTTP headers X-Developer-Id and X-Api-Key. These will be provided by SnapMD developers. You will begin with sandbox keys, and be provided with separate production keys when your app is approved.

Tokens

Authentication in the API is handled through OAuth2 bearer token authentication. To retrieve a token, you POST a login resource to the API. Any activity on behalf of a user must use this method of authentication.

POST v2/account/token

application/json, text/json

Sample: 
POST https://sandbox.connectedcare.md/api/v2/account/token HTTP/1.1
User-Agent: Fiddler
Content-Type: application/json
X-Developer-Id: ******
X-Api-Key: ******

{ 
  'userTypeId': 1, // 1: patient/customer, 2: clinician/admin
  'hospitalId' : 1,
  'email' : <email>,
  'password' : <password>
}

Response Format

application/json, text/json

Sample: 
{
  "data": [
    {
      "access_token": <token>
    }
  ],
  "total": 1
}

GET api/hello

We have our version of Hello World that you can use to verify things are set up correctly.

Request Information

application/json, text/json

Sample: 
GET https://sandbox.connectedcare.md/api/hello  HTTP/1.1
User-Agent: Fiddler
Content-Type: application/json
Authorization: Bearer ***********
X-Developer-Id: ******
X-Api-Key: ******

The value of "hello" is the username associated with the token; Response body:

application/json, text/json

Sample: 
{ "hello" : string }
